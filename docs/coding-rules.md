# コーディングルール

## 基本

- 既存のコードスタイルに合わせる
- 可読性を優先する
- 必要以上に設計を変更しない

## バックエンド

- Fat Controllerを避ける
- ビジネスロジックはServiceへ集約する
- ModelをControllerから直接操作しすぎない

## フロントエンド

- コンポーネントを適切に分割する
- 状態管理をシンプルに保つ
- API通信を共通化する

## 命名

- 命名は既存実装に合わせる
- 略語はできるだけ使わない

## 品質

- 未使用コードを残さない
- 不要なコメントは書かない
- エラー処理を実装する
